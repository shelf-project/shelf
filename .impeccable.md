# Shelf — Design Context

> Captured by `/teach-impeccable` for the impeccable skill set. This file is the
> source of truth for every later design decision in `shelfd/ui/`. If a future
> change can't be defended against the **Design Principles** below, cut it.

## Design Context

### Users

Shelf has three audiences, in descending operational urgency:

1. **Trino-on-Kubernetes operators** (us, DevOps, on-call). Open the UI when a
   replica's hit rate sags, when shelfd OOMKills, when Foyer's LODC drops
   spike, or when a pin-list change needs to be verified inside 60 seconds.
   Lives in dense data — the `Live`, `Hot tables`, `Lab`, and `Admin` tabs.
2. **Leadership / PM / new joiners**. Land on the URL once a quarter (or
   their first week). Have ~5 seconds before they form an opinion. The
   `Story` tab — and only the Story tab via `#public` — is for them. They
   should walk away with a one-sentence answer: *"Shelf saves us $X / month
   and Y TB of S3 reads."*
3. **OSS evaluators**. shelfd is Apache-2.0 at `github.com/shelf-project/shelf`.
   When someone clones the repo and runs the smoke harness, the first
   screen they see is the Story tab. It's also the project's storefront.

### Brand Personality

Three words: **library-quiet · earned · alive**.

- **Library-quiet** — the chrome is calm, neutral slate, no neon, no
  glassmorphism. Whisper, don't shout.
- **Earned** — every pixel justifies itself with a number. No decoration,
  no gradient text, no sparkline-as-jewellery. The skill's "AI Slop Test"
  is the bar: if a stranger could mistake any panel for "default Vercel
  starter dark mode," it's wrong.
- **Alive** — the cache is a *physical place*. Books arrive on a shelf,
  capacity rises like liquid in a glass, the shader background ripples on
  every hit. Motion encodes meaning, never decoration.

The product is *literally* called **shelf**. The metaphor — desk (DRAM),
shelf (NVMe), library (S3) — is the brand. Every overdrive moment must
either reinforce that metaphor or get cut.

### Aesthetic Direction

A refined, minimal **frame** around dense, technical **interiors**:

- **Frame**: low-chrome neutral slate (existing palette in `styles.css`
  lines 11–58 stays). Only chromatic accents are the OK / WARN / ERR
  traffic lights — they're how operators read health at a glance.
- **Interior**: information-dense, Stephen-Few-bullet-chart-grade panels.
  Tight grouping inside cards, generous whitespace between them.
- **Library-as-software**: the bookshelf, the desk, the now-serving feed,
  the reading-room ambient shader. References: `Stripe Atlas` (calm
  density), `Linear` (precise spring physics), `Apple HIG` (depth via
  motion not chrome). Anti-references: `Datadog` (chart soup),
  `Grafana out-of-the-box` (plain, joyless), default `shadcn/ui` AI-slop.
- **Theme**: dark by default to match Grafana operator workflow; light
  variant equally polished — not an afterthought.

### Design Principles

These four bind every panel, every animation, every component decision.

1. **Every pixel passes the "good or bad?" test.** Operators have ~3
   seconds per panel. Fixed thresholds, traffic-light colour, the
   answer is on screen without reading prose. (Codified for Grafana in
   `~/trino/AGENTS.md`; same rule applies here.)
2. **Motion encodes meaning.** A book jiggles → cache hit. A book falls →
   eviction. A vessel ripples → fresh poll. A row morphs places →
   leaderboard shift. If you can remove the animation and the meaning
   survives, the animation is decoration — cut it.
3. **One moment of poetry per tab.** The skill warns: *"focus creates
   impact, excess creates noise."* Each tab gets exactly one signature
   overdrive moment (Story = shader + physics shelf, Live = capacity
   vessels, Hot = view-transition swap, Lab = WebGL heat field, Admin =
   HRW ring viz). The rest stays calm so the moment lands.
4. **Reduced-motion is a first-class theme, not a fallback.** Every
   shader, physics scene, and view transition has a hand-tuned static
   counterpart that's still beautiful — never just "no animation."
   `prefers-reduced-motion` is checked once via `useReducedMotion`, and
   nothing skips the gate.
